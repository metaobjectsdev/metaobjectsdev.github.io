Cross-language metadata standard · Apache 2.0

The metadata layer for AI-first development.

Define typed entities once. Generate idiomatic code across TypeScript, Java, .NET, and Python. Use the same metadata at runtime. Catch drift in the type checker. The metamodel is the durable spine; generated code is the disposable artifact.

Read the spec → GitHub

Three pillars

Codegen

Emit idiomatic per-language code from a single metadata model — Drizzle/Zod + Fastify for TS, JOOQ/Spring for Java, Pydantic/FastAPI for Python. Hand-edit-preserving regeneration via three-way merge.

Runtime metadata

Load metadata at runtime and drive behavior dynamically — CRUD, validation, relationships, dynamic admin UIs, LLM tool registration. Less code in the repo; less surface area for AI to drift on.

Drift detection

Catch divergence between code and metadata before it ships. Compile-time breakage in the type checker when schemas change underneath the logic that depends on them.

30-second example

Define an entity in metadata. Generate the types, persistence, and validation around it. Use the same metadata at runtime.

// metaobjects.config.ts
import { defineMetadata } from '@metaobjects/metadata';

export default defineMetadata({
  entities: {
    Order: {
      fields: {
        id:        { type: 'uuid', primaryKey: true },
        customerId: { type: 'uuid', required: true },
        amount:    { type: 'decimal', scale: 2 },
        status:    { type: 'enum', values: ['pending', 'paid', 'shipped'] },
      },
    },
  },
});
# Generate code across all supported languages
$ meta gen --target=typescript
$ meta gen --target=java

# Drift check against existing code
$ meta check
